What is the Document Object Model (DOM)?

The document Object Model is a programming language that allows interaction with the webpage's elements. The DOM can be visualized as a upside down tree model where The tree's root is Document. It represent Entire HTML document. Every other elements in the HTML is a node and they are parts of the Document.
A Website use each Nodes or the elements like <h1> <p>as a object datatype.

How do you select an element from the DOM?

We can select an Element from the DOM in several ways.
1. Select by ID:
To select an element by its id, we use the document.getElementById() method.Example, document.getElementById("myId").
2. Select by Class:
To select elements by their class name, we use document.getElementsByClassName(). Example: document.getElementsByClassName
("myClass").
3. Select by Tag:
To select elements by their tag name, we use document.getElementsByTagName(). Example: document.getElementsByTagName("p").
4. Select by CSS Selector:
To select elements by their CSS selector, we use document.querySelector(). Example: document.querySelector("#myId").
5. Select by Multiple Elements:
To select multiple elements by their tag name, we use document.querySelectorAll(). Example: document.querySelectorAll("p").

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique used in the Document Object Model (DOM) to handle events efficiently. It allows users to attach a single event listener to a parent element instead of attaching a listener to each individual child element. This reduces the number of event listeners and improves performance.

The main advantage of event delegation is that it improves code reusability and reduces memory usage. When an event occurs on the parent element, the event handler function is triggered, and the event object contains information about the specific element that triggered the event.

Event delegation is especially useful when dealing with dynamic content, such as adding or removing elements dynamically. By attaching a single event listener to the parent element, user can handle all the events triggered by the child elements, reducing the number of event listeners and improving efficiency.
